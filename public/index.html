Home Assistant Database Analysis Tool

Database Cleanup Guide

⚠️ Important:

Before performing any cleanup operations, create a backup of your Home Assistant configuration!

Step 0: Create a Backup

Visit Home Assistant Backups and create a manual backup before proceeding.

Step 1: Understanding the Recorder

The recorder in configuration.yaml controls what data is saved to the database: 

recorder:
  purge_keep_days: 30
  db_url: !secret recorder_db_url
  exclude:
    entity_globs:
      - "sensor.weather_*"
    domains:
      - automation
      - script
⚠️ Warning:

Avoid using include in the recorder config. Once you use include, only entities mentioned as included will be recorded, all other will now be excluded by default.

See the official Recorder documentation for full reference

Use the Generate YAML feature in the Entities and Domains tabs to help configure exclusions based on storage usage analysis.

Step 2: Understand what takes space in your database

Navigate to the tabs of this plugin to understand what takes some space in your database. You can sort and filter to understand your entities.

  • Tables: gives an overview of your database table size. You can get details by hovering on the column title.
  • Entitied: this main screen gives you insights about your entities. The sizes shown are estimated on rows average size, so you will see a small difference between estimation and actual database size.
  • Integrations and Domains: sums entities by integrations and domains to get more understanding.

Step 3: Purge Historical Data

Go to Home Assistant Developer Tools → Actions and search for recorder.purge:

  1. Uncheck "Keep days" checkbox (or this may remove some ata you wanted to keep)
  2. Check "Repack" checkbox (this defragments the database)
  3. Leave "Exclude devices" unchecked unless you have specific devices to exclude
  4. Check "Apply filter" to apply recorder config exclusions
  5. Click "Call Service"
  6. Restart Home Assistant after the purge completes

Note: The purge operation only removes historical states, not statistics.

Step 4: Clean Up Statistics

Since recorder.purge doesn't remove statistics, use the Statistics Tool:

  1. Click "Select" button
  2. Choose "Select all problematic statistics"
  3. Review the list of selected entities carefully
  4. Click "Delete selected statistics"
  5. Confirm the deletion

Step 5: Final Cleanup

To further optimize the database:

  1. Run recorder.purge one more time with the same settings
  2. Restart Home Assistant again
  3. Refresh this tool to verify the database size reduction
✓ Success:

Your database should now be significantly smaller! Monitor the storage and adjust your recorder exclusions based on what's consuming the most space.

Troubleshooting:
  • If the database size doesn't change much, you may be excluding important entities. Review your recorder config.
  • Some integrations may create entities automatically. Check the Integrations tab to identify heavy consumers.
  • Consider excluding entire domains (like automation, script) that generate many state changes.
